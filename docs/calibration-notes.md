# スコア検証・カリブレーション ノート

初回検証時点：2026-07-03（実データ運用 開始直後）。以後、データ蓄積に合わせて更新する。

## 1. 検証で分かったこと

### 合成スコアの分布（health check）
`history.topix.json`（855営業日, 2023-01 〜 2026-07）を集計：

| 指標 | 値 |
| --- | --- |
| 中央値 | 49.9 |
| 平均 | 48.0 |
| 分布 | min 0.2 / p25 30 / p75 65 / max 99 |
| バンド滞在率 | 極度の恐怖20% / 恐怖24% / 中立14% / 貪欲28% / 極度の貪欲14% |

→ **中央値≒50・平均≒48 で系統的な偏りがなく、5バンドに幅広く分布**。正規化設計
（percentile / anchor / 反転）は意図どおり機能している。特定バンドへの張り付きもなし。

### 最大の注意点：ヒストリカルの「採用指標数（coverage）」
各営業日の採用指標数の分布：

| coverage | 日数 |
| --- | --- |
| 1 | 376 |
| 2 | 201 |
| 3 | 1 |
| 4 | 70 |
| 5 | 99 |
| 6 | 106 |
| 7 | 2 |

→ **過去の大半（約7割）が 1〜2 指標のみで算出**されている。これは各指標の
increment キャッシュ（P/C・騰落・新高安の全銘柄終値・週次信用残 など）が遡れる範囲が
限られ、古い日付ほど #1 モメンタム中心になるため。**8指標フル（coverage 7〜8）は
直近の数日のみ**で、長期のヒストリカルは「フル合成の Fear & Greed」ではなく
「少数指標（主にモメンタム）の正規化」に近い。

つまり **長期チャートの前半を “本来の合成スコア” として解釈してはいけない**。
これはバグではなくデータ蓄積の途上であり、日次運用が続くほど coverage が上がり、
低 coverage の期間は既定表示（1年）の窓から外れていく。

### 個別指標の妥当性（直近フル coverage 日）
raw → score の対応は config 定義どおりで矛盾なし（抜粋・2026-07-02 TOPIX版）：

- モメンタム raw +6.98% → 28（percentile。直近1年では上方乖離が小さめ＝やや弱気側）
- 日経VI raw 33.78 → 20（高VI＝恐怖、反転が効いている）
- P/C raw 2.38 → 8（プット厚め＝ヘッジ需要大、反転）
- 空売り比率 raw 43.47% → 2（高い空売り＝弱気、反転）
- 信用評価損益率 raw −3.42% → 83（アンカー -10→50 / 0→100 の線形補間＝82.9。整合）

→ 反転フラグ・アンカー・パーセンタイルはいずれも正しく作用。**現時点で修正が必要な
mis-calibration は検出されず**。

## 2. 今回の対応
- **透明性の改善**：ヒストリカルチャートのツールチップに coverage を表示し、
  低 coverage 期間がある場合は注記を出す（少数指標で算出している旨）。
  → 前半の低 coverage スコアを「フル指数」と誤読させない。
- 数値パラメータ（アンカー・重み・lookback）は **変更しない**。1〜2点の実測で
  アンカーを動かすのは過剰最適化になるため。

## 3. データ蓄積待ちの残作業（次回以降）
- **#6 信用評価損益率の tier-3 較正**：松井の実測点が現状 **1点のみ**。数十点貯まったら
  tier-2（週次信用残MTM近似）との重複期間で bias/scale を評価し、必要なら較正式を
  オフセットのみ→線形回帰に拡張。
- **#3 新高値新安値**：52週窓が充填され coverage が安定的に 8 になったら、
  フル coverage 期間だけで各指標スコアの相関チェック（次元横断の冗長性確認）。
- **合成スコアの再評価**：フル coverage の履歴が数十営業日たまった時点で、
  分布・バンド滞在率・イベント整合（急落局面で恐怖側へ振れるか）を再検証。
- 必要に応じてアンカー（騰落レシオ 70/100/130、信用評価損益率 -30/-20/-10/0）を
  日本市場の実分布に合わせて微調整。
